## Start Session

Kick off a new working session. Do the following steps in order:

1. Run `git fetch && git pull` to sync with the remote
2. Look for and read any of the following documents if they exist — check both the project root and the parent directory, as docs are sometimes kept one level up:
   - `SESSIONS.md` or `SESSION.md` — developer diary / session notes
   - Any `*-PRD.md`, `*-TechSpec.md`, or `*-Spec.md` files
   - `README.md` — project overview
3. **Start local servers** — check `package.json` for defined dev scripts (e.g. `dev`, `start`, `studio`). If any are defined and not already running, start them in the background. Let Mike know which ones were started and what ports they're on.
4. Greet Mike with a 2–3 sentence summary of where the project left off and what the immediate next step is, based on what you found
